람다식(Lambda Expressions)은 **코드를 짧고 간단하게 쓰기 위한 문법**입니다.

특히 “함수형 인터페이스”와 함께 사용됩니다.

---

# ✅ 람다식이란?

> **메서드를 한 줄의 표현으로 줄여 쓴 것**

즉, 이름 없는 메서드를 빠르게 표현하는 방식입니다.

---

# ✅ 기존 방식 vs 람다식

### ✅ 1) 기존 방식 (익명 클래스 사용)

```java
Runnable r = new Runnable() {
    @Override
    public void run() {
        System.out.println("Hello");
    }
};
r.run();
```

너무 길죠? 😅

### ✅ 2) 람다식 사용

```java
Runnable r = () -> System.out.println("Hello");
r.run();
```

같은 기능인데 훨씬 짧고 깔끔해졌습니다.

---

# ✅ 람다식의 기본 형태

```
(매개변수) -> { 실행할 코드 }
```

예:

```java
(x, y) -> { return x + y; }
```

하지만 코드가 짧으면 더 줄일 수 있습니다:

```java
(x, y) -> x + y
```

---

# ✅ 예시 ① (매개변수 없음)

```java
() -> System.out.println("Hi");
```

---

# ✅ 예시 ② (매개변수 1개)

```java
x -> x * x
```

---

# ✅ 예시 ③ (매개변수 여러 개)

```java
(a, b) -> a + b
```

---

# ✅ 예시 ④ (여러 줄 코드)

```java
(x, y) -> {
    int sum = x + y;
    return sum;
}
```

---

# ✅ 왜 쓰는가?

### ✅ 코드를 짧고 읽기 쉽게 만든다

불필요한 익명 클래스 반복 코드 제거

### ✅ 함수형 프로그래밍 스타일 지원

Java에서도 함수 같은 기능을 쉽게 전달 가능

### ✅ 컬렉션 처리에 매우 유용

Stream API와 함께 강력하게 활용

---

# ✅ 람다가 동작하기 위한 조건

> 람다식은 “함수형 인터페이스”에만 적용할 수 있다.

즉, 추상 메서드가 1개인 인터페이스여야 한다.

예: `Runnable`, `Comparator`, `Callable`, `Function` 등

---
Lambda 표현식의 문법(Syntax)

람다식은 기본적으로 다음 형태를 따릅니다:

---

# ✅ 람다식의 기본 형태

```
(매개변수) -> { 실행 코드 }
```

이게 가장 기본 모양입니다.

---

# ✅ 1) 매개변수 부분

* 0개일 수도 있고
* 1개일 수도 있고
* 여러 개일 수도 있음

예:

```
()          // 매개변수 없음
x           // 매개변수 1개 (괄호 생략 가능)
(a, b)      // 매개변수 2개 이상
```

---

# ✅ 2) 화살표 `->`

이건 **“왼쪽은 입력, 오른쪽은 실행 코드”**라는 뜻입니다.

---

# ✅ 3) 실행 코드 부분

* 중괄호 `{ }` 안에 여러 줄을 넣을 수도 있고
* 한 줄이면 중괄호 생략 가능

예:

```
{ System.out.println("Hi"); }
```

또는

```
System.out.println("Hi")
```

---

# ✅ Syntax를 패턴별로 정리

## ✅ A. 매개변수 없음

```
() -> System.out.println("Hello");
```

## ✅ B. 매개변수 1개, 타입 생략 가능

```
x -> x * x
```

(원래는 `(int x) -> x * x`라고 쓸 수도 있지만 보통 생략함)

## ✅ C. 매개변수 여러 개

```
(a, b) -> a + b
```

## ✅ D. 실행 블록이 여러 줄이면 {}

```
(x, y) -> {
    int sum = x + y;
    return sum;
}
```

---

# ✅ 타입을 생략해도 컴파일러가 알아서 추론

Java의 특징 중 하나:

> 매개변수 타입을 생략해도 된다!

예:

```
(int x, int y) -> x + y
```

이런 식으로 타입을 명시할 수도 있지만 대부분 이렇게 씀:

```
(x, y) -> x + y
```

컴파일러가 자동으로 "x와 y는 어떤 타입인지" 체크해줍니다.

---

# ✅ 초보자를 위한 람다 문법 요약표

| 키워드   | 설명             |
| ----- | -------------- |
| `()`  | 매개변수 부분        |
| `->`  | 입력에서 실행 코드로 연결 |
| `{ }` | 여러 줄 실행 코드     |

---

# ✅ 전체적으로 보면 이렇게 생각하면 쉬움

> 왼쪽(매개변수)은 **입력**,
> 오른쪽은 **실행할 내용**.

---

# ✅ 초보자용 한 줄 정리

> 람다식은 `(입력) -> 실행` 형태로, 메서드를 간단하게 표현하는 문법이다.

---

# ✅ 단일 expression이면 return 생략 가능

람다식에서 실행 코드가 **한 줄짜리 expression(식)**이면, Java는 다음을 자동으로 합니다:

* expression을 계산하고
* 그 값을 그대로 **자동으로 리턴(return)**

즉, `return` 키워드를 직접 쓰지 않아도 됩니다.

---

# ✅ 예시 1: 단일 expression (자동 return)

```java
(x, y) -> x + y
```

이렇게 쓰면 Java는 내부적으로 이렇게 이해합니다:

```java
(x, y) -> {
    return x + y;
}
```

---

# ✅ 예시 2: 한 줄 표현식에서 return 생략됨

```java
x -> x * x
```

Java는 자동으로

```java
x -> {
    return x * x;
}
```

처럼 동작합니다.

---

# ✅ 하지만 여러 줄이면 return 필요함!

이 경우에는 return을 **직접** 써야 합니다:

```java
(x, y) -> {
    int sum = x + y;
    System.out.println(sum);
    return sum;   // 반드시 return 필요!
}
```

여러 줄이므로 `{ }`가 필요하고, return도 직접 적어야 함.

---

# ✅ 한 줄 vs 여러 줄 비교

### ✅ 단일 expression

```
(x, y) -> x + y
```

➡ 자동 return

### ✅ 여러 줄

```
(x, y) -> {
    System.out.println(x + y);
    return x + y;
}
```

➡ return 직접 필요

---

# ✅ 초보자용 한 줄 정리

> 실행 코드가 "딱 한 줄 표현식"이면 return을 생략할 수 있다.
> 여러 줄이면 직접 return을 적어야 한다.
